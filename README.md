# 大搜渠道利润测算模型 V1.0.0 (Profit Model)

这是一个专门用于**大搜渠道业务**（以亲子鉴定无创/个人/司法为例）的单页 React 动态利润边界测算工具。

该系统通过精密的成本分摊基准和前端可视化交互，能够帮助运营人员和负责人快速测算不同投放流量、薪酬分配、转化漏斗以及退款率等变动要素下的**盈亏平衡点边界和利润上限**。

## 🎯 核心能力

1. **动态联动测算核心**：调整“综合单成本”、“售中提成率”或“无创客单价”等任一细小参数，整表（包含总营收、总利润、ROI 及各类成本占比）立刻重算更新。
2. **多业务线独立配置**：支持按业务属性独立切分（无创/个人/司法等），并为各自定制转化漏斗和部门经理专属成本。
3. **策略矩阵模拟器 (Matrix Simulator)**：基于设定的客单价及成本跨度，能一键自动计算生成不同规模日均量下的净利润阵列图，并在内部自动标亮出 🎯 **盈亏平衡临界点**。
4. **AI 首席财务官智能洞察**：无缝对接了 Google Gemini 2.5 API ，随时针对当前各项参数截面生成通俗易懂的盈利结构诊断与优化建议。
5. **本地快照与报表导出**：支持记录不同的组合打法模型作为“历史快照”进行对比，且能够一键全量导出为 CSV/Excel 进行线下传递分析。

## 🏗️ 架构说明 (Architectural Refactoring)

> **注**：本项目经过了一次彻底的解耦重构。所有的原 `app.jsx` 单一文件功能已被拆分为规范化的前沿 React Hooks 式工程。虽然**未做任何业务逻辑层或者底层计算逻辑的调整修改**，但整体获得了极高的应用扩展性和可维护性。

项目的目录结构核心划分为以下 4 个平面：

- **`/src/constants/`** —— 静态资源与常量层 (集中管理了初始化的各数值状态 `initialState.js`)
- **`/src/utils/`** —— 工具与核心计算层 (包含不依赖 React 生命周期的财务核心算式 `calculators.js` 与 `formatters.js`)
- **`/src/hooks/`** —— 数据与状态管理层 (专职处理 React 的数据流与生命周期，如 `useProfitModel.js`、`useGeminiAI.js` 等)
- **`/src/components/`** —— 视图与组件层 (UI 呈现视图层，负责装配各种子组件如 `ResultTable.jsx`、`HistoryPanel.jsx`)

主入口 `src/app.jsx` 现仅作为顶层状态分发的 Glue（粘合剂）容器。

## 🚀 快速启动

本仓库使用 Vite 作为极速构建工具。

### 1. 环境准备
确保您的计算机已安装 [Node.js](https://nodejs.org/) (建议通常使用 LTS 版本)。

### 2. 安装依赖
在项目根目录下执行：
```bash
npm install
```

### 3. 本地开发服务器启动
启动带有本地热重载（HMR）的测试环境：
```bash
npm run dev
```
之后在浏览器中打开命令行提示的如 `http://localhost:5173/` 网址即可。

> **登录授权提示**：出于保护数据的目的，体验 Demo 前需验证。**开发测试默认密码为：`666666`**。如果需要变更，请调整 `src/components/LoginScreen.jsx` 中的逻辑判定或移入外置环境变量。

### 4. 生产环境构建打包
当需要部署到线上环境（例如 Nginx，Vercel 或者 GitHub Pages 时），执行：
```bash
npm run build
```
编译产物将会统一构建输出在 `/dist` 目录中，直接向服务器挂载该目录即可。

## 💡 Gemini API 密钥
目前 AI 生成洞察服务需要在系统中调用外网的 Gemini 引擎。如有二次开发变动需求或需要将服务暴露外网，请自行将 `src/hooks/useGeminiAI.js` 中调用的常量环境变量化为安全注入。
